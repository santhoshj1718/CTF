Introduction to Cryptography

Cryptography is the practice of secure communication in the presence of adversaries. In CTFs, crypto challenges typically involve:

  • Breaking weak encryption implementations
  • Exploiting poor key management
  • Identifying patterns in encoded data
  • Side-channel attacks
  • Mathematical attacks on cryptographic primitives

Common Crypto Challenge Types

Type Description Examples
Classic Ciphers Historical encryption methods Caesar, Vigenère, Substitution
Modern Symmetric Block and stream ciphers AES, DES, RC4
Asymmetric Public-key cryptography RSA, ECC, Diffie-Hellman
Hashing One-way functions MD5, SHA1, bcrypt
Encoding Data representation Base64, Hex, ASCII
Steganography Hiding data in other data LSB, Audio/Image hiding

Basic Terminology

  • Plaintext: Original message
  • Ciphertext: Encrypted message
  • Key: Secret used for encryption/decryption
  • Encryption: Process of converting plaintext to ciphertext
  • Decryption: Process of converting ciphertext to plaintext
  • Cryptanalysis: Study of breaking cryptographic systems
Note: Many CTF crypto challenges involve recognizing patterns and applying known attacks rather than implementing complex cryptanalysis.

Classic Ciphers

Historical ciphers that are often vulnerable to frequency analysis and other attacks.

Caesar Cipher

A substitution cipher that shifts letters by a fixed number down the alphabet.

# Encryption: E(x) = (x + key) mod 26 # Decryption: D(x) = (x - key) mod 26 # Example with key=3 (ROT3) Plaintext: HELLO Ciphertext: KHOOR
# Python implementation def caesar(text, shift): result = "" for char in text: if char.isupper(): result += chr((ord(char) + shift - 65) % 26 + 65) elif char.islower(): result += chr((ord(char) + shift - 97) % 26 + 97) else: result += char return result # Brute force all rotations for i in range(26): print(f"{i}: {caesar(ciphertext, -i)}")

ROT13 - Special case of Caesar cipher with shift=13

# Linux command echo "Uryyb Jbeyq" | tr 'A-Za-z' 'N-ZA-Mn-za-m' # Output: Hello World # Python import codecs codecs.encode("Hello World", 'rot13')

Vigenère Cipher

A method of encrypting alphabetic text by using a simple form of polyalphabetic substitution.

# Using keyword "KEY" Plaintext: ATTACKATDAWN Key: KEYKEYKEYKEY Ciphertext: KFAKQKXFAWFE # Encryption: C_i = (P_i + K_i) mod 26 # Decryption: P_i = (C_i - K_i) mod 26
# Python implementation def vigenere(text, key, decrypt=False): result = [] key_len = len(key) for i, char in enumerate(text): if char.isalpha(): shift = ord(key[i % key_len].upper()) - 65 if decrypt: shift = -shift new_char = chr((ord(char.upper()) - 65 + shift) % 26 + 65) result.append(new_char) else: result.append(char) return ''.join(result)
Note: The Vigenère cipher can be broken using:
  • Kasiski examination (to find key length)
  • Frequency analysis (once key length is known)

Substitution Cipher

Each letter in the plaintext is replaced by a letter corresponding to a fixed system.

# Example substitution key A -> Q, B -> G, C -> T, D -> F, E -> L, F -> X, G -> J, H -> M, I -> K, J -> W, K -> E, L -> Y, M -> P, N -> R, O -> D, P -> S, Q -> Z, R -> U, S -> O, T -> V, U -> A, V -> I, W -> N, X -> H, Y -> B, Z -> C Plaintext: CRYPTO Ciphertext: TUYVSD
# Frequency analysis approach 1. Identify single-letter words (likely 'A' or 'I') 2. Look for common patterns (e.g., 'TH', 'HE', 'ING') 3. Use frequency tables (E is most common in English) 4. Look for doubled letters (common: LL, SS, EE, TT, etc.)

CTF Tip: For classic cipher challenges:
  • Look for patterns in the ciphertext (repeated sequences, letter frequencies)
  • Try online solvers first (like quipqiup, dcode.fr)
  • Check if it's a combination of ciphers (e.g., ROT13 then substitution)
  • If all else fails, brute force with all possible keys

Modern Cryptography

Contemporary encryption algorithms used in real-world systems, often with CTF-specific vulnerabilities.

Symmetric vs. Asymmetric

Symmetric Asymmetric
Keys Single shared key Public/private key pair
Speed Fast Slow
Examples AES, DES, ChaCha20 RSA, ECC, ElGamal
CTF Attacks Weak keys, ECB mode, IV reuse Small exponents, shared primes

Block Cipher Modes

Mode Description Vulnerabilities
ECB Each block encrypted independently Pattern leakage, block shuffling
CBC Chained with previous ciphertext Padding oracle, IV reuse
CTR Counter mode (turns into stream) Nonce reuse
GCM Authenticated encryption Nonce reuse

Common Vulnerabilities

# ECB mode - identical plaintext blocks produce identical ciphertext # Can reveal patterns and allow block reordering # CBC with predictable IV - allows first block manipulation # Especially dangerous if IV = 0 or IV = key # Padding oracle - error messages reveal padding validity # Can decrypt ciphertext byte by byte # Nonce/IV reuse in CTR/GCM - breaks confidentiality # Two-time pad vulnerability
CTF Tip: For modern crypto challenges:
  • Check for weak parameters (small keys, predictable IVs)
  • Look for error messages that might reveal information
  • Try to find mathematical weaknesses (in RSA, ECC, etc.)
  • Consider side-channel attacks (timing, error-based)

XOR Encryption

XOR (exclusive OR) is a fundamental operation used in many cryptographic systems.

Properties of XOR

# Basic properties: A ⊕ A = 0 A ⊕ 0 = A A ⊕ B = B ⊕ A (A ⊕ B) ⊕ C = A ⊕ (B ⊕ C) # If C = A ⊕ B, then: A = B ⊕ C B = A ⊕ C

Single-byte XOR

# Encrypting 'HELLO' with key 0x3C H (0x48) ⊕ 0x3C = 0x74 E (0x45) ⊕ 0x3C = 0x79 L (0x4C) ⊕ 0x3C = 0x70 L (0x4C) ⊕ 0x3C = 0x70 O (0x4F) ⊕ 0x3C = 0x73 Ciphertext: 0x74 0x79 0x70 0x70 0x73 ("typps")
# Python brute force def single_byte_xor(ciphertext): for key in range(256): plaintext = bytes([b ^ key for b in ciphertext]) if b' the ' in plaintext: # Common English word check return plaintext, key return None, None

Multi-byte XOR (Repeating Key)

# Using key 'SECRET' Plaintext: ATTACK AT DAWN Key: SECRE TSECRET Ciphertext: ?*?*?* ?* ?*?*
# Finding key length (Kasiski examination) 1. Look for repeated sequences in ciphertext 2. Measure distances between them 3. Key length is likely a divisor of these distances # Once key length is found, solve as multiple single-byte XORs

XOR in Modern Cryptography

# AES uses XOR in its round functions # Stream ciphers like RC4 use XOR with keystream # One-time pad is theoretically unbreakable when: 1. Key is truly random 2. Key is same length as plaintext 3. Key is never reused

xortool - Analyze multi-byte XOR cipher

# Install pip install xortool # Analyze file xortool ciphertext.bin # Brute force with known characters xortool -l 5 -c " " ciphertext.bin
CTF Tip: For XOR challenges:
  • Check if it's single-byte or multi-byte XOR
  • Look for patterns that might reveal the key length
  • Try known plaintext attacks if part of the message is known
  • Use frequency analysis on each key position

RSA Cryptography

RSA is a widely used public-key cryptosystem based on the difficulty of factoring large integers.

RSA Basics

# Key generation 1. Choose two large primes p and q 2. Compute n = p * q (modulus) 3. Compute φ(n) = (p-1)*(q-1) 4. Choose e such that 1 < e < φ(n) and gcd(e, φ(n)) = 1 5. Compute d = e⁻¹ mod φ(n) (private key) # Encryption: c = mᵉ mod n # Decryption: m = cᵈ mod n

Common RSA Attacks

# Small modulus - can be factored # Small e (e=3) with small message - cube root attack # Same n with different e - common modulus attack # Same e with different n - Chinese Remainder Theorem # Wiener's attack - when d is small # Boneh-Durfee - when d < n^0.292

CTF RSA Challenges

# Given n, e, c - factor n to get p and q # Given p and q - compute φ(n) and d # Given multiple ciphertexts with same modulus # Given partial key information (like d modulo φ(n))
# Python solution template from Crypto.Util.number import inverse, long_to_bytes n = 123... # modulus e = 65537 # public exponent c = 456... # ciphertext # If p and q are known p = ... q = ... phi = (p-1)*(q-1) d = inverse(e, phi) m = pow(c, d, n) print(long_to_bytes(m))

Advanced RSA Attacks

# Franklin-Reiter related message attack # When two messages satisfy m1 = f(m2) mod n # Coppersmith's short pad attack # When messages are related and padding is short # LSB oracle attack # When server reveals if decrypted message is even/odd # Bleichenbacher's attack # PKCS#1 v1.5 padding oracle

CTF Tip: For RSA challenges:
  • First check if n can be factored (factordb.com)
  • Look for small exponents (e=3, e=1)
  • Check for common modulus or related messages
  • If given multiple ciphertexts, consider CRT

AES Encryption

Advanced Encryption Standard (AES) is a symmetric block cipher used worldwide.

AES Basics

# Key sizes: 128, 192, or 256 bits # Block size: 128 bits # Rounds: 10 (128-bit), 12 (192-bit), or 14 (256-bit) # Operations: - SubBytes (S-box substitution) - ShiftRows (row shifting) - MixColumns (column mixing) - AddRoundKey (XOR with round key)

Common AES Vulnerabilities

# ECB mode - identical plaintext blocks produce identical ciphertext # CBC with predictable IV - allows first block manipulation # Padding oracle - error messages reveal padding validity # Key reuse - same key with different modes/IVs # Side-channel attacks - timing, power analysis

Padding Oracle Attack

# When server reveals if padding is valid # Can decrypt ciphertext without knowing key # Works against CBC mode with PKCS#7 padding # Steps: 1. Modify ciphertext blocks to guess padding bytes 2. Use server responses to determine plaintext 3. Repeat for all blocks
# Python example using pwntools from pwn import * def oracle(c): r = remote('target', 1234) r.send(c.hex()) resp = r.recvline() r.close() return b"success" in resp # Implement padding oracle attack...

padbuster - Automated padding oracle tool

# Usage padbuster http://target/decrypt?c=XXXX XXXX 8 -encoding 0 -error "Invalid padding"
CTF Tip: For AES challenges:
  • First identify the mode of operation
  • Check if IV is predictable or reused
  • Look for error messages about padding
  • Try bit-flipping attacks in CBC mode

# Rainbow tables - precomputed hash chains # Dictionary attacks - common passwords # Collision attacks - two inputs with same hash # Length extension - for Merkle-Damgård hashes # Example: MD5 collision # Two different files with same MD5 hash

Encoding Schemes

Methods for representing data in different formats (not encryption).

Common Encoding Types

Type Description Example Recognition Pattern
Base64 6-bit chunks to ASCII (A-Z,a-z,0-9,+,/,=) SGVsbG8= Ends with = or ==, alphanumeric with +/
Hex Binary to hex digits (0-9,a-f) 48656c6c6f Even length, only 0-9 and a-f
ASCII Codes Decimal character codes 72 101 108 108 111 Numbers 32-126 separated by spaces
URL Encoding Percent-encoded characters %48%65%6c%6c%6f % followed by two hex digits
Binary Base-2 representation (0-1) 01001000 01100101... Groups of 8 bits (sometimes space-separated)
Morse Code Dots and dashes .... . .-.. .-.. --- Combination of . and - with separators

Encoding/Decoding Techniques

# Base64 (Linux) echo "Hello" | base64 # Encode echo "SGVsbG8=" | base64 -d # Decode # Hex (Python) "Hello".encode('hex') # Encode "48656c6c6f".decode('hex') # Decode # ASCII codes (Python) ' '.join(str(ord(c)) for c in "Hello") # Encode ''.join(chr(int(c)) for c in "72 101 108 108 111".split()) # Decode # URL encoding (Python) import urllib.parse urllib.parse.quote("Hello World!") # Encode urllib.parse.unquote("%48%65%6c%6c%6f") # Decode
