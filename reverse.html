Introduction to Reverse Engineering

Reverse engineering involves analyzing compiled programs to understand their functionality without access to source code.

x86/x64 Assembly

Understanding assembly is crucial for effective reverse engineering.

Registers

32-bit (x86) registers:

EAX - Accumulator EBX - Base ECX - Counter EDX - Data ESI - Source Index EDI - Destination Index ESP - Stack Pointer EBP - Base Pointer EIP - Instruction Pointer

64-bit (x64) extended registers:

RAX, RBX, RCX, RDX - 64-bit versions R8-R15 - Additional registers RSP - Stack Pointer RBP - Base Pointer RIP - Instruction Pointer

Common Instructions

Instruction Description Example
MOV Move data MOV EAX, 42
PUSH/POP Stack operations PUSH EAX
CALL/RET Function calls CALL 0x401000
JMP/Jcc Jumps JNZ 0x401020
LEA Load effective address LEA EAX, [EBX+4]

Calling Conventions

# cdecl (32-bit Linux) - Args pushed right-to-left - Caller cleans stack - Return in EAX # System V AMD64 (64-bit Linux) - Args in RDI, RSI, RDX, RCX, R8, R9 - Stack for additional args - Return in RAX # stdcall (Windows) - Args pushed right-to-left - Callee cleans stack - Return in EAX

Function Prologue/Epilogue

# Typical function prologue push ebp ; Save old base pointer mov ebp, esp ; Set new base pointer sub esp, 0x10 ; Allocate stack space # Typical function epilogue mov esp, ebp ; Restore stack pointer pop ebp ; Restore base pointer ret ; Return
CTF Tip: Key patterns to recognize:
  • Function prologues/epilogues
  • String comparisons (often before jumps)
  • Mathematical operations (key calculations)
  • Stack adjustments (local variables)

ARM Assembly

ARM architecture is common in mobile and embedded systems.

Registers

R0-R12 - General purpose R13 (SP) - Stack Pointer R14 (LR) - Link Register R15 (PC) - Program Counter CPSR - Current Program Status Register

Common Instructions

Instruction Description Example
MOV Move data MOV R0, #42
LDR/STR Load/store LDR R0, [R1]
PUSH/POP Stack operations PUSH {R0,R1}
B/BL Branch/Branch with Link BL 0x1000
CMP Compare CMP R0, R1

Conditional Execution

# Instructions can be conditional ADDEQ R0, R1, R2 ; Add if equal CMP R0, #10 ; Compare MOVGT R0, #1 ; Move if greater than

Calling Convention

# ARM (32-bit) - Args in R0-R3 - Stack for additional args - Return in R0 # ARM64 (AArch64) - Args in X0-X7 - Stack for additional args - Return in X0

ARM Debugging Tools

QEMU - ARM emulation

# Run ARM binary on x86 qemu-arm ./arm_binary # Debug with gdb-multiarch qemu-arm -g 1234 ./arm_binary gdb-multiarch -q (gdb) target remote :1234

  2. Behavior Observation:
    ./crackme # Try input: test # Observe "Wrong password" message
  3. Disassembly:
    gdb crackme (gdb) disassemble main
  4. Key Findings:
    • Password comparison at 0x401152
    • Input length check (must be 10 chars)
    • XOR operation with 0x42
  5. Solution:
    # Find XOR-encoded password in binary # Decode with: bytes(x ^ 0x42 for x in encoded)

Practice Techniques

Training Methodology:
  1. Static Analysis First:

    Always start with static analysis before running the binary

  2. Document Findings:

    Keep notes of function addresses, interesting strings, and potential vulnerabilities

  3. Dynamic Verification:

    Use debugging to verify your static analysis findings

  4. Automate Repetitive Tasks:

    Create scripts for common operations (decoding, bruteforcing)

Reverse Engineering CTF Tips

General Strategy

  1. Run file and strings
  2. Check for packing with binwalk
  3. Open in disassembler (Ghidra/IDA)
  4. Identify key functions (main, check, verify)
  5. Analyze algorithm or patch binary

Common CTF Patterns

Pattern Solution
Simple compare Patch jump or change compare value
Mathematical transform Reverse the algorithm
Obfuscated code Dynamic analysis with debugger
Packed binary Dump from memory or find unpack routine

Challenge-Specific Tips

Crackme Challenges

  • Look for "success" and "failure" strings
  • Trace user input flow through the program
  • Patch conditional jumps to bypass checks
  • Find hardcoded comparisons with your input

Packed Binaries

  • Look for unusual entry point code
  • Use memory dumping techniques
  • Find the Original Entry Point (OEP)
  • Look for unpacking loops

Malware Challenges

  • Monitor file system changes
  • Check for suspicious process injection
  • Analyze network traffic
  • Look for persistence mechanisms

Quick Wins

  • Search for "flag{" or "CTF{" in strings
  • Look for XOR operations with single byte keys
  • Check for hardcoded passwords
  • Patch conditional jumps (JNZ → JZ)

Time-Saving Techniques

# Automate string extraction rabin2 -z binary > strings.txt # Quick function graph r2 -AAA -qc "ag > /tmp/graph.dot" binary xdot /tmp/graph.dot # Find cross-references to interesting strings grep -B5 -A5 "flag{" strings.txt
