Introduction to Forensics

Digital forensics involves investigating digital devices to find, analyze, and present evidence. In CTFs, forensics challenges typically involve:

  • Analyzing file formats and metadata
  • Recovering deleted or hidden data
  • Examining memory dumps
  • Investigating network traffic
  • Solving steganography challenges
  • Analyzing disk images
  • Extracting mobile device data
  • Timeline analysis of system events

Common Forensics Challenge Types

Type Description Example Techniques Common Tools
File Analysis Examining file contents and metadata File carving, magic numbers, strings binwalk, xxd, exiftool
Memory Forensics Analyzing RAM dumps Process analysis, network connections, registry Volatility, Rekall
Network Forensics Examining network traffic PCAP analysis, protocol dissection Wireshark, tshark
Steganography Finding hidden data LSB, spectrograms, file carving steghide, stegsolve
Disk Forensics Analyzing disk images Partition recovery, file system analysis Autopsy, The Sleuth Kit

Forensic Methodology

1. Identification: Recognize potential sources of evidence 2. Preservation: Create forensic copies (never work on original) 3. Analysis: Examine evidence using appropriate tools 4. Documentation: Record findings and methods 5. Presentation: Summarize conclusions # CTF-Specific Approach: 1. Identify the challenge type (file, memory, network, etc.) 2. Use appropriate tools for initial analysis 3. Look for anomalies or hidden data 4. Extract potential flags or interesting artifacts 5. Verify findings and document the process 6. If stuck, try alternative approaches or tools

Forensic Principles

# Order of Volatility (most to least volatile) 1. CPU registers, cache 2. Memory (RAM) 3. Network connections 4. Running processes 5. Disk storage 6. Remote logging and monitoring data 7. Physical configuration, network topology 8. Archival media # Forensic Soundness Requirements: - Evidence must be preserved without alteration - All actions must be documented - Methods must be repeatable - Analysis must be verifiable
Note: Many forensics challenges involve multiple steps - the flag might be hidden behind several layers of extraction or decoding.

File Carving Techniques

# Using foremost (carves based on headers/footers) foremost -i disk.img -o output_dir foremost -t jpg,pdf,zip -i file.bin -o output # Specific types # Using binwalk (signature based) binwalk -e file.bin # Auto-extract binwalk -D 'zip archive:zip' file.bin # Extract specific type binwalk -Me file.bin # Recursive extraction # Manual carving with dd dd if=file.bin of=extracted.jpg bs=1 skip=1234 count=5678 # Using photorec (for deleted files) photorec /dev/sdX photorec /d output_dir /f file_to_recover # Using scalpel (configurable carving) scalpel -c /etc/scalpel.conf -o output file.bin
# Carving workflow: 1. Identify potential file types in the binary (binwalk, strings) 2. Note the offsets of potential file headers 3. Determine the file size (either from metadata or search for footer) 4. Extract the file using dd or specialized tools 5. Verify the extracted file integrity 6. Repeat for all potential files

Advanced File Analysis Techniques

# XOR Analysis (for obfuscated files) xorsearch -s file.bin "flag{" xorstrings file.bin # Find strings in XORed files # Entropy Analysis (identify encrypted/compressed regions) binwalk -E file.bin # Entropy graph ent file.bin # Calculate entropy (0-8 scale) # File Difference Analysis cmp -l file1.bin file2.bin # Byte-by-byte comparison xxd file1.bin > file1.hex xxd file2.bin > file2.hex diff file1.hex file2.hex # Malware Analysis Tools peframe malware.exe # PE file analysis rabin2 -zz malware.bin # Extract strings and more

File Analysis Tools

binwalk - Firmware analysis tool

# Install sudo apt install binwalk # Basic usage binwalk file.bin # Analyze file signatures binwalk -e file.bin # Extract known file types binwalk -Me file.bin # Recursive extraction binwalk -E file.bin # Entropy analysis binwalk -W file1.bin file2.bin # Compare files # Advanced options: binwalk --matryoshka --depth=5 file.bin # Deep recursive scan binwalk -A firmware.bin # Architecture analysis binwalk -Y file.bin # Fast analysis mode

xxd - Hex dump viewer

# View hex dump xxd file.bin | less xxd -g 1 file.bin # Single-byte grouping xxd -s 0x100 -l 256 file.bin # Show 256 bytes at offset 0x100 # Search for patterns xxd file.bin | grep "504b" # Search for ZIP header xxd -p file.bin | tr -d '\n' | grep -o "504b.*" # Extract from offset # Convert hex back to binary xxd -r hexdump.txt > recovered.bin

foremost - File carving tool

# Install sudo apt install foremost # Basic usage foremost -i disk.img -o output_dir foremost -t jpg,pdf,doc -i file.bin -o output # Configuration /etc/foremost.conf # Define custom file headers/footers
CTF Tip: For file analysis challenges:
  • Always check file headers and footers first
  • Look for embedded files or hidden data
  • Try multiple tools - sometimes one misses what another finds
  • Check both the beginning and end of files for anomalies
  • Don't ignore file metadata (EXIF, PDF properties, etc.)
  • If a file appears corrupted, try fixing headers or removing garbage data
  • For password-protected files, try common passwords or empty passwords

Memory Forensics

Analyzing memory dumps to find processes, network connections, and hidden data.

Volatility Framework

# Basic Volatility commands (version 2.x) volatility -f memory.dmp imageinfo # Identify profile volatility -f memory.dmp --profile=Win7SP1 pslist # List processes volatility -f memory.dmp --profile=Win7SP1 pstree # Process tree volatility -f memory.dmp --profile=Win7SP1 netscan # Network connections volatility -f memory.dmp --profile=Win7SP1 filescan # Scan for files volatility -f memory.dmp --profile=Win7SP1 dumpfiles -Q 0x1234 -D output/ # Dump file volatility -f memory.dmp --profile=Win7SP1 memdump -p 1234 -D output/ # Dump process memory # Volatility 3 commands (new version) vol.py -f memory.dmp windows.info vol.py -f memory.dmp windows.pslist vol.py -f memory.dmp windows.netscan vol.py -f memory.dmp windows.dumpfiles --physaddr 0x1234 -D output/

Memory Analysis Workflow

1. Identify the memory profile (imageinfo) 2. Check running processes (pslist, pstree) - Look for unusual process names or paths - Check for processes with no parent or unusual parents 3. Examine network activity (netscan, connscan) - Look for unusual connections - Check for listening ports 4. Scan for files (filescan, dumpfiles) - Look for interesting file names or extensions 5. Check for malicious activity (malfind, apihooks) - Look for injected code or hooked functions 6. Dump suspicious processes (memdump, procdump) - Analyze dumped memory with strings or other tools 7. Check registry (hivelist, printkey) - Look for persistence mechanisms - Check for unusual registry values

Network Forensics

Analyzing network traffic captures to find hidden information.

PCAP Analysis Basics

# Basic tshark commands tshark -r capture.pcap # Summarize traffic tshark -r capture.pcap -Y "http" # Filter HTTP traffic tshark -r capture.pcap -T fields -e frame.number -e ip.src -e ip.dst # Show IPs tshark -r capture.pcap -z io,phs # Protocol hierarchy tshark -r capture.pcap -qz conv,ip # IP conversations tshark -r capture.pcap -qz conv,tcp # TCP conversations # Extract files from pcap foremost -i capture.pcap -o output/ binwalk -e capture.pcap

Common Network Forensics Tasks

Task Tool/Command Purpose
Follow TCP Stream Wireshark: Right-click → Follow → TCP Stream Reconstruct conversations
Extract Files binwalk, foremost, tcpxtract Recover transferred files
Analyze DNS tshark -Y "dns" Examine DNS queries
Find Credentials strings capture.pcap | grep -i "pass" Search for passwords
Export Objects Wireshark: File → Export Objects Extract HTTP, SMB files
GeoIP Lookup tshark -r capture.pcap -Y "ip" -T fields -e ip.src | geoiplookup Geolocate IP addresses

Advanced PCAP Analysis

# Extract SSL/TLS keys for decryption tshark -r capture.pcap -o "ssl.keylog_file:sslkeys.log" -qz follow,ssl,ascii,1 # Reassemble fragmented packets editcap -F pcap capture.pcap reassembled.pcap # Filter specific protocols tshark -r capture.pcap -Y "ftp || http || smb || dns" # Extract DNS queries tshark -r capture.pcap -Y "dns" -T fields -e dns.qry.name # Analyze VoIP calls tshark -r capture.pcap -q -z rtp,streams

CTF Forensics Tips

General Strategy

  • Start with basic analysis (file, strings, binwalk)
  • Look for hidden files or data
  • Check metadata (exiftool, pdfinfo)
  • Try multiple tools - they often find different things
  • When stuck, think about what you haven't checked yet
  • Document your process - helps when you get stuck
  • Take breaks - fresh eyes often spot what you missed

Common Flag Locations

# Typical flag formats: flag{Th1s_1s_4_fl4g} FLAG{Th1s_1s_4_fl4g} CTF{Th1s_1s_4_fl4g} THM{Th1s_1s_4_fl4g} HTB{Th1s_1s_4_fl4g} # Common hiding places: - File metadata (EXIF, PDF properties) - Hidden files in disk images - Unusual network traffic - Memory artifacts (process memory, network connections) - Steganography in images/audio - Deleted files or slack space - Alternate data streams (Windows) - Database records (SQLite, registry)

Quick Checks

  • Run strings on every file
  • Check for embedded files with binwalk
  • Look at hex headers/footers with xxd
  • Search for flag formats with grep -a "flag{" file
  • Try common stego tools on image/audio files
  • Check for password-protected archives (try empty password)
  • Examine all metadata with exiftool
